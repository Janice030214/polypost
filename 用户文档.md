# Polypost 使用说明

> 这是一个**帮你把博客文章一键发到官网**的小工具。
> 把文章粘进去，按几个按钮，11 种语言自动翻译、自动同步、自动发布。本来要花一两个小时手工填的内容，现在 5 分钟搞定。

---

## 一、它是干什么的？

平时往官网发一篇博客，你大概要手工做这些事：

| 手工要做的事 | 大概多久 |
|---|---|
| 进 Strapi 后台，新建文章，填标题、描述、作者、日期、slug | 5 分钟 |
| 把文章里的图片一张一张上传到媒体库 | 5-10 分钟（看图多少） |
| 上传封面、选分类 | 1 分钟 |
| 一个一个语种点 "Translate with AI" 翻译 | 10-20 分钟 |
| 检查每个语种翻得对不对、给每张图填 alt 文本 | 10 分钟 |
| 一个一个语种点 Publish 发布 | 5 分钟 |
| **总计** | **半小时到一小时** |

用 Polypost 做同样的事，**只需要这几步**：

1. **粘贴**文章
2. **拖一张封面**
3. 点 **处理文章** → **一键翻译** → **同步到 Strapi** → **一键发布**

完事。

---

## 二、怎么打开它？

在浏览器（**Chrome / Safari / Edge 都行**）的地址栏，输入：

```
http://localhost:4000
```

按回车。看到一个**淡紫色**的网页，标题是「Strapi 博客上传 `</>`」就对了。

> 如果打不开（提示无法访问），说明工具的"服务"没启动。**找开发的同事帮你启动一下**就行。

## 四、发布一篇全新文章（最常见）

### 第 1 步：粘贴文章

1. 在 Lark 里打开你写好的文章
2. **全选**（Cmd+A 或 Ctrl+A）→ **复制**（Cmd+C 或 Ctrl+C）
3. 回到 Polypost 页面，在「① 文章内容」的空白区里**粘贴**（Cmd+V 或 Ctrl+V）

粘贴后你会看到文章在里面显示出来，**带格式、带图片**。

### 第 2 步：上传封面

在「② 封面 cover」那块：

- **点击** 那个虚线框 → 选一张图
- 或者**直接把图片拖**进虚线框

成功的话能看到封面的缩略图。

> 如果你的文章里第一张图就能当封面，**这一步可以跳过**，工具会自动用第一张正文图作封面。

### 第 3 步：检查分类和元数据

在「③ 分类与元数据」：

- **分类**：默认选了「AI Usage & Technical Guides」(一般教程)。如果你这篇是**新模型预告**，点「AI Model Updates」；如果是**和竞品对比、案例**，点「Business Case Studies」。
- 下面的**标题、描述、作者、日期、slug、metaTitle** 这些小字段——**全部可以不填**！工具会从文章里自动识别。如果识别得不对，再手动改这里。

### 第 4 步：选要发布的语言

在「④ 选择要发布的语言」：

- 默认 11 个全部勾上（en + 简中 + 繁中 + 日 + 韩 + 德 + 法 + 西 + 葡 + 俄 + 阿）
- 想全部发，**不用动**
- 只想发英文，点「只 en」
- 想全选回来，点「全选 11 个」

### 第 5 步：按顺序点 4 个按钮 ⭐⭐⭐

到「⑤ 操作」区，按顺序点这 4 个按钮：

#### ① 点「📦 处理文章」

工具会：
- 把文章里的图片**搬到我们自己的服务器**（避免链接失效）
- 上传你刚才传的封面
- 自动识别文章里的标题、描述等
- 调 AI **自动给每张图写英文 alt 描述**

**等待时间**：5-30 秒（图多就慢一点，AI 识图是大头）

完成后会出现一个 **「正文图片宽度」面板**，列出文章里的每一张图。

#### ② 调整每张图的宽度和 alt（在两个按钮之间）

「正文图片宽度」面板里，每张图你能看到：

```
[图片预览]  Alt 文本：[AI 已填]
            [输入框：A diagram of...]    [64%] [80%] [100%] [自定义]
            链接
```

- **Alt 文本**：AI 自动写好了（蓝底显示），不满意你可以**直接改**
- **宽度**：默认 64%。如果某张图要大一点 → 点 `80%` 或 `100%`；要自定义 → 在「自定义」框输入，比如 `420px`

调好后再点下一个按钮。

> 💡 **一定要先调好图片宽度再翻译**！翻译之后再改虽然也可以，但翻译前调好，所有语种的图都会按你的选择走。

#### ③ 点「🌐 一键翻译」

工具会**并行**翻译所有勾选的语言（除了 en）。

**等待时间**：10 个语言一起翻，大概 5-15 秒。

成功后状态栏会显示一排小绿标 ✅：

```
✅ 翻译完成
已就绪：en zh zh-Hant ja ko de fr es pt ru ar
```

#### ④（可选）点「🧹 清除错误格式」

文章如果有这些"小毛病"，点这个按钮自动清理：

- 文字里的 `灰底代码格式`（小段落像代码一样的灰底，不是真正的代码块）
- 文章里**蓝色的 API 链接**（比如 `https://api.atlascloud.ai/v1` 显示成蓝色超链接，你不想要）
- 段落之间莫名其妙的空行
- H2 / H3 标题前面的空行规整一下
- 图片紧贴文字（不留空行）

**这一步可选**，如果你没看到这些问题，可以直接跳到下一步。

> 这个按钮**点完会自动同步**到后台，所以你可以不用再点「同步到 Strapi」。

#### ⑤ 点「📤 一键同步到 Strapi」

工具会在 Strapi 后台**创建 11 种语言的草稿**（默认草稿，没发布）。

**等待时间**：5-10 秒。

成功后会有个**直达 Strapi 后台的链接**，你可以点进去看一眼内容对不对。

#### ⑥ 点「🚀 一键发布所有语言」

**确认草稿没问题后**，点这个按钮一次性发布全部 11 个语种。

**等待时间**：3-5 秒。

完成后会返回**所有 11 个官网链接**，可以一一点开看效果：

```
✅ en → https://www.atlascloud.ai/blog/guides/your-slug
✅ zh → https://www.atlascloud.ai/zh/blog/guides/your-slug
✅ ja → https://www.atlascloud.ai/ja/blog/guides/your-slug
... (11 条)
```

**完成！** 这一篇文章就上线了。

## 五、修改已经发过的文章

经常会有这种情况：发出去的文章发现 typo 了、想换张封面、想加几段、某个翻译不准等。用工具的「编辑模式」很方便。

### 第 1 步：切到编辑模式

在最上面「模式」那一栏，点 **「编辑已有（覆盖）」**。

下面会出现一个输入框：「文章 documentId 或后台 URL」。

### 第 2 步：找到这篇文章的"地址"

去 Strapi 后台，打开那篇你要改的文章。

**复制浏览器地址栏**里的整段网址，类似：
```
https://strapi-admin.atlascloud.ai/admin/content-manager/collection-types/api::blog.blog/cn0auliepg7c7yb6c5wmah0m?plugins[i18n][locale]=en
```

回到 Polypost，把这串地址**粘到输入框**里，点 **[加载]**。

### 第 3 步：工具会把后台的现有内容**全部回填**到表单里

- 「① 文章内容」里**回填整篇正文**（可以直接在里面编辑）
- 「② 封面」显示当前封面
- 「③ 分类与元数据」全部字段填好
- 「正文图片宽度」面板显示**每张图当前的宽度和 alt**
- 「④ 语言」自动勾上**该文章已有的所有语种**

### 第 4 步：按你的需求改

#### 场景 A：只想改某个字段（比如改个标题、改个描述）

1. 直接在「③ 分类与元数据」里改对应字段
2. 直接点 **「📤 一键同步到 Strapi」**（跳过处理和翻译，因为正文没动）
3. 点 **「🚀 一键发布所有语言」**

#### 场景 B：想换封面

1. 在「② 封面」点「移除封面」→ 拖入新封面
2. **「📤 一键同步到 Strapi」**
3. **「🚀 一键发布所有语言」**

#### 场景 C：想改正文内容（加段落、改文字、改图）

1. 在「① 文章内容」里直接**修改文字**
2. **重新**点 **「📦 处理文章」**（让工具重新整理新内容）
3. （如果有新图）调整新图的宽度和 alt
4. **「🌐 一键翻译」**（让其他语种跟上新内容）
5. **「📤 一键同步到 Strapi」**
6. **「🚀 一键发布所有语言」**

#### 场景 D：想给已发布的文章**加一个新语种**

例如这篇原来只发了 en + zh，现在想再加日语：

1. 加载文章（en + zh 会自动勾上）
2. 在「④ 语言」**额外勾上** `ja`
3. **「🌐 一键翻译」**
4. **「📤 一键同步到 Strapi」**
5. **「🚀 一键发布所有语言」**

## 六、一些有用的小技巧

### 翻译之前先看看转得对不对

点 **「预览（不上传）」** 按钮（在「① 文章内容」下面），工具会显示一个 markdown 预览，让你看转出来的样子。**不会真的上传任何东西**。看完关掉就行。

### 改图片大小

在「正文图片宽度」面板，每张图右边有按钮：
- `64%`（默认，居中显示，留两侧白边）
- `80%`（占大部分宽度）
- `100%`（铺满文字宽度）
- `自定义`：输入任意值，例如 `420px`、`50%`、`auto`

### 图片说明（alt 文本）

每张图的输入框可以编辑。AI 自动写的是英文的简短描述（约 12 词以内）。

- 觉得不准 → 直接改
- 觉得啰嗦 → 改短
- 不想要 alt → 清空

> **小知识**：alt 文本是给搜索引擎和盲人读屏看的，**写好对 SEO 有帮助**。

### 不想用 AI 写 alt，全部自己写

让开发同事在 `.env` 里清掉视觉模型配置即可。之后所有 alt 输入框会空着等你填。

### 切换其他视觉模型

点右上角 **⚙️ 视觉模型设置**，下拉里选一个：

| 模型 | 速度 | 质量 |
|---|---|---|
| qwen/qwen3-vl-30b-a3b-instruct（推荐）| 快（每张约 3-5 秒）| 好 |
| qwen/qwen3-vl-235b-a22b-instruct | 中等 | 更好 |
| qwen/qwen3-vl-8b-instruct | **最快** | 一般 |
| zai-org/glm-5v-turbo | 快 | 不错 |
| moonshotai/kimi-k2.6 | **慢**（约 30 秒一张）| 思考更深，但太慢 |
| 自定义 | - | 自己填模型名 |

设置后会**记住**（保存在浏览器里），下次打开还是这个。

### 「清除错误格式」做了什么？

它清理这些常见问题：
- 文字里那种**灰底像代码一样**的小段（除非是代码块）
- `api.atlascloud.ai` 这种 API 地址自动变蓝色超链接 → 改回普通文字
- 段落之间多余空行 → 删掉
- 大标题（H2）前面少了空行 → 加上
- 小标题（H3）前面多了空行 → 删掉
- YouTube 视频前后留空行
- 图片前后紧贴文字
- 两张图挨在一起 → 中间留空行
- 表格前后紧贴文字

**应用到所有语种**，**点完自动同步**到 Strapi。

### 只想发某一个语言

在「④ 选择要发布的语言」点 **「只 en」**，所有其他语种取消。然后按 4 步流程走。

### 用 YouTube 视频做封面

在「③ 分类与元数据」找到 `coverYoutubeId` 那个框，填 YouTube 视频 ID（视频网址里 `?v=` 后面那串字符，例如 `s8CIBscdEHI`）。

封面就会变成视频缩略图 + 播放按钮。

## 七、遇到这些情况怎么办？

### 状态条变红色 + 报错

工具底部那个状态条会显示 ✅ 绿色（成功）/ 🔴 红色（失败）/ 🟣 紫色（加载中）。

#### 「翻译失败：HTTP 401」之类

意思是：**登录状态过期了**。
- 解决：**再点一次「一键翻译」** 就行（工具会自动重新登录）
- 如果还是失败：找开发同事检查 admin 账号密码

#### 「X 张正文图未能搬运」

意思是：**Lark 私链的图工具拉不到**（Lark 的图需要登录才能看）。

解决办法：
1. 在 Lark 里打开那张图 → **右键另存到电脑**
2. 回到 Polypost，**直接拖**这张图到「① 文章内容」对应位置
3. 重新点 **「📦 处理文章」**

#### 「请先粘贴内容」

意思是「① 文章内容」是空的，先粘文章再点按钮。

#### 「没有封面」

意思是没传封面、文章里也没图、`coverYoutubeId` 也没填。**三选一**搞定。

### 按钮点不动（灰色）

按钮是**按顺序解锁**的：
- 「🌐 一键翻译」要先点过「📦 处理文章」才会亮
- 「📤 同步」要先点过「🌐 一键翻译」才会亮（编辑模式下加载后直接亮）
- 「🚀 发布」要先点过「📤 同步」才会亮

如果点了上一步还是灰的，**刷新页面**重试。

### 图片预览框是空白 / 显示「无法预览」

通常**不影响功能**——只是浏览器加载缩略图被防盗链拦了。点图片下面那个链接（在新标签打开）能看到图就说明图是正常的。

### 编辑模式加载失败

错误：「加载失败」或者「HTTP 404」

- 检查你粘的网址/编号是不是这篇文章的
- 如果只复制了文章 documentId（24 位字符的那串），也可以直接粘进去

### 已经发布了，但想撤回

工具**没有撤回按钮**。要撤回需要**手动**：
1. 进 Strapi 后台 → 找到那篇文章
2. **每个语种**分别点 [Unpublish]

### 改完之后线上没变

如果文章已经发布过：
- 你在工具里改完点了「同步到 Strapi」，文章状态会变成 **"Modified"**（已发布但有改动）
- **必须再点一次「🚀 一键发布所有语言」**，新内容才会让线上生效

### 浏览器报错 "Failed to parse URL"

你**不是用 Chrome/Safari/Edge 打开的 http://localhost:4000**。可能是用了什么"预览面板"或者打开了 .html 文件本身。请到正常浏览器地址栏输入 `http://localhost:4000`。

## 八、一些重要提醒 ⚠️

1. **顺序很重要**：必须先「📦 处理文章」→ 再「🌐 一键翻译」→ 再「📤 同步」→ 再「🚀 发布」。跳步会失败。

2. **图片宽度要在翻译前调好**：翻译之前调，所有语种都会按你选的来。翻译之后再调虽然也行，但麻烦。

3. **编辑老文章一定走「编辑已有」模式**，否则会**新建一篇重复的文章**。

4. **改完一定要点同步 + 重新发布**：光在工具里改不算数，必须把改动**推回 Strapi** 才行；如果文章本来就是已发布状态，**还得重新点发布**才能让线上生效。

5. **同步用的是 PUT 覆盖**：在工具里改完同步，会**完全覆盖**后台里这个语种的草稿。所以如果你想保留某个语种已经手工改好的版本，**别在工具里改完同步**——这样会把那个手工改的版本盖掉。

6. **改了模型设置 / 切换浏览器后**：视觉模型设置存在**当前浏览器**里。换浏览器或者清了缓存要重新设。

7. **AI 翻译不是 100% 完美**：发布前**最好抽几个语种**到 Strapi 后台快速过一眼，看看有没有奇怪的翻译。

## 九、5 分钟新手速通

完整流程压缩版，新人**照着做一遍**就会：

```
1. 浏览器输入 http://localhost:4000

2. Lark 里全选文章 → 复制 → 在「① 文章内容」粘贴

3. 拖一张图到「② 封面」（或者跳过让工具用第一张图）

4. 「③ 分类」一般默认即可；其他字段不填

5. 「④ 语言」一般默认全选

6. 「⑤ 操作」按顺序点：
   📦 处理文章       （等 5-30 秒）
   → 调每张图宽度 + 改 alt
   🌐 一键翻译       （等 5-15 秒）
   🧹 清除错误格式   （可选，建议点一下）
   📤 一键同步到 Strapi  （等 5-10 秒）
   🚀 一键发布所有语言   （等 3-5 秒）

7. 拿到 11 条官网链接，挑几个打开看下效果

完成！
```

## 十、有问题找谁？

- **工具打不开 / 报错排查 / 加新功能** → 找开发同事
- **AI 翻译效果不好 / 想换模型** → 自己点 ⚙️ 视觉模型设置切换，或找开发同事改 Strapi 翻译插件配置
- **发现 Bug** → 截图 + 描述发给开发同事
